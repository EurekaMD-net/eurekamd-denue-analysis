### Claude Agent SDK cutover (commit `f506a5e`)

Per operator directive "use SDK for the demo, just as Jarvis", swapped
the Sage Anthropic provider from `@anthropic-ai/sdk` (Messages API,
requires `ANTHROPIC_API_KEY` env) to `@anthropic-ai/claude-agent-sdk`
(authenticates via `~/.claude/.credentials.json` — Max Plan + Extra

call_endpoint / draft_sql / decline as Zod-typed MCP tools the model
must pick from. Narrative path uses bare `query()` with diff-based
text-delta streaming.

### Supabase login + JWT bearer (commit `3d032c7`)

Replaced the single-shared-`X-Api-Key` gate with email+password login
backed by self-hosted Supabase Auth (`db.mycommit.net`). Browser users
get a JWT in `Authorization: Bearer …`; machine clients keep using
`X-Api-Key` (coexisting paths). Hand-rolled HS256 JWT verification in
`bearer-auth.ts` (Node Web Crypto, no jsonwebtoken dep) with in-memory
verify-cache (5-min TTL, LRU-tail at 10k entries).

Auth audit (3 parallel agents — crypto / middleware decision / frontend
session) ran post-ship; verdict was FAIL/PASS-WITH-FIXES/FAIL.

### Auth audit fixes (commit `256eea2`)

5 Critical + 11 Warnings closed in one bundle:

- A C1: `aud`/`role` missing-claim bypass (truthy short-circuit) → strict `typeof === "string"` checks
- A C2: `exp` NaN bypass → `Number.isFinite(claims.exp)` guard before arithmetic
- A W1: cache grows past MAX_SIZE when nothing's expired → LRU-tail 10% drop
- A W2: `alg=none` / `alg=RS256` confusion → JWT header parsed first; HS256-only enforced
- A W3: header regex too permissive → base64url chars only + `.trim()` input
- A R3: `sub` strict-type check
- B W1: `Bearer`/`bearer` case-sensitivity unified
- B W2: `SUPABASE_JWT_SECRET` trim + boot-time length warning
- C C1: MapShell still sent JWT as `X-Api-Key` → fixed to `Authorization: Bearer`
- C C2: Sign-out token leak → store has `abortRegistry`; SageMode registers stream; `signOut()` walks + aborts
- C C3: TanStack Query cache survived sign-out → store carries QueryClient; `signOut()` runs `cancelQueries()` + `clear()`
- C W1: 9-file variable rename `apiKey → accessToken` + selector subscription fix
- C W4: error translations for network/CORS/Supabase rate-limit variants

Backend tests: 988 passing (up 14 from new strict-claim defenses).

### Sage live end-to-end + SQL gate fixes (commit `a0dad46`)

First real client-end query (`demo@eurekamd.cloud` → JWT → Caddy →
Sage SQL fallback → COFEPRIS state-grain rollup) surfaced two
defensive-too-aggressive gate bugs:

1. **psql command tags pollute output**. `psql -tA -c <multi-statement>`
   emits `BEGIN\nSET\nSET\n…\nCOMMIT` lines around the EXPLAIN JSON /
   COPY CSV. `JSON.parse` choked; CSV parser saw `{BEGIN: SET}` rows.
   Fix: `-q` (quiet) flag on both EXPLAIN and execute psql calls.

2. **FORBIDDEN_RELATIONS was overbroad on EXPLAIN plan check**. The
   list included small base tables (e.g. `cofepris_farmacias`, 2,381
   rows) behind allowlisted views. Planner inlines views → EXPLAIN
   reflects the base table → false rejection. Fix: split into two
   lists. `FORBIDDEN_RELATIONS` (textual SQL gate, unchanged) +
   `FORBIDDEN_SEQ_SCAN_RELATIONS` (narrow: only the 4 huge tables —
   `establecimientos`, `sesnsp_delitos_municipal_raw`,
   `censo_ageb_raw`, `censo_iter`).

Live-verified query: NL/CDMX top-5 farmacias, $0.0588 total, 34s
wall-clock. Narrative voice landed correctly; Sonnet 4.6 spotted the
overlapping-endorsement-flag issue in the data unprompted.
