## What's in the live DB

### Establecimientos (the spine)

`establecimientos` — 6,097,681 rows × wide schema. Backfilled join keys:

- `entidad` (CVE_ENT, 2-char) — 100% populated
- `area_geo` (CVE_MUN_5, 5-char ENT||MUN) — backfilled from CLEE chars 1-5
- `sector_actividad_id` / `subsector_actividad_id` / `rama_actividad_id` /
  `subrama_actividad_id` / `clase_actividad_id` (SCIAN 2/3/4/5/6-digit) —
  backfilled from CLEE chars 6-11
- `ageb` (13-char CVEGEO ENT+MUN+LOC+AGEB) — backfilled via PostGIS spatial
  join against `ageb_polygons`. 99.99975% coverage (15 unmatched are bad
  lat/lon).

Indexes: btree on entidad, area_geo, clase_actividad_id, sector_actividad_id;
GIST on geom; GIN on nombre.

### v0.2.2 Health infrastructure (CLUES, this commit batch)

| View / Table                      | Rows   | What it gives you                                                                                                                                                                               |
| --------------------------------- | ------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `clues_raw` (table, 68 TEXT cols) | 63,708 | Verbatim DGIS CLUES catálogo (ENERO 2026 cut, 8 instituciones, 41k EN OPERACION + 22k FUERA + ~60 construcción)                                                                                 |
| `clues` (mat. view, 20 cast cols) | 41,381 | Filtered to EN OPERACION. cve_mun + cve_loc + nivel_atencion::int + lat/lon + geom POINT(4326). 39,946 (96.5%) geocoded. GIST on geom + btree on cve_mun, cve_loc, institucion, nivel_atencion. |

**Join keys**:

- `cve_mun` (5-char) — joins to censo_municipios, coneval_pobreza_municipal, coneval_irs_municipal, establecimientos.area_geo

## Mat-view refresh cadence

Several analytics endpoints read from materialized views: `mv_sector_grade_matrix`,
`mv_national_treemap`, `mv_delitos_municipal_yearly`, `mv_mortalidad_municipal_yearly`.
After a loader run, counts drift from the source tables until the views
are refreshed.

**Auto-refresh shipped 2026-05-06**: `denue-matview-refresh.timer` (daily
04:00 UTC + 5min jitter, Persistent=true) wraps `scripts/refresh-matviews.sh`.
Test run: 89s for all 4 MVs. Logs via `journalctl -u denue-matview-refresh -f`.

Manual refresh (post-loader, before waiting for the timer):

```
./scripts/refresh-matviews.sh
# or for the full v0.3 perf bootstrap:
docker exec -i supabase-db psql -U postgres -d postgres < scripts/perf-matviews.sql
```

The DDL is idempotent (DROP+CREATE = full refresh). Backend handlers fall
back to live SQL on `relation does not exist`, so analyzer keeps working
on a fresh DB that hasn't run the migration — just 100-150× slower.
