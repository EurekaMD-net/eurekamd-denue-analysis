# v0.3.1 — Frontend Hardening Plan

**Status**: Phases 1–4 SHIPPED 2026-05-11. Phase 5 (RH-5 backend extension) pending.
**Predecessor**: v0.3 demo shipped 2026-05-10 (`a0dad46`)
**Scope**: Close all R-tier deferreds from R2 (frontend UX) and C (auth flow) audits. No new mode features. No backend changes (server is closed).

## Mission

Take the v0.3 demo from "client-ready surface" to "client-handed-over surface". Every R-tier finding becomes either a fix or an explicit "won't fix because…" with a written reason. Bundle size drops. Cross-tab sign-out works. Filter pins do what they advertise.

**Anti-mission**: no new modes, no new data layers, no Datatur ingestion. Pure hardening.

### Phase 2 — Auth hardening (S items, ~2 hours) ✅ SHIPPED 2026-05-11

RH-10, RH-11, RH-12, RH-13 all closed in code. Cross-tab sign-out wires via supabase-js's native SIGNED_OUT broadcast (which already piggy-backs on storage events) + a belt-and-suspenders direct storage listener for externally-cleared tokens.

**Audit**: single qa-auditor pass — verdict **PASS WITH WARNINGS** (2 critical, 4 warnings):

- **C1 (real regression caught)** — first-pass `SAFE_QUERY` regex `/^[A-Za-z0-9._=&%-]*$/` rejected `! ~ * ' ( )`, which `encodeURIComponent` leaves un-encoded per RFC 3986 unreserved set. A user searching `Domino's` or `(Sucursal)` in `useSearch` would have gotten `bad_path 400` from the frontend before the call left the tab. **Fixed**: extended SAFE*QUERY to `/^[A-Za-z0-9.*=&%~!_'()-]_$/`. Regression test pinned in `web/src/api/client.test.ts` covering 7 representative passthrough chars.
- **C2 (comment drift)** — store comment claimed "listener short-circuits if session is already null" but listener calls cleanupLocal unconditionally. The flow is still correct (cleanupLocal is idempotent — that's the actual invariant), but the comment misled. **Fixed**.
- **W2** (positive/negative test mixed) — split into two `it` blocks.
- **R5** (regex DRY) — exported `SUPABASE_TOKEN_KEY_RE` from LoginGate; store test now imports the real one instead of recreating it.
- **W4** (token-refresh storage event amplification) — added inline comment explaining the cost shape.
- **W1** (session_loading is forward-looking plumbing) and **W3** (no LoginGate-render test) deferred — see deferreds below.

**Numbers**: 3 source files + 2 new test files, +247 / −20. 78 web tests (up from 52: +26 across `client.test.ts` and `store.test.ts`). Backend test count unchanged at 948. Total vitest 1026 ✓. Production build green; new index chunk `index-CdT-4Sgc.js` live; `/api/search?q=Domino%27s` returns 401 (auth) not 400 (frontend rejection), confirming C1 fix.

**Deferred (with justification)**:

- **W1** — `session_loading` code path has no production consumer today (LoginGate gates all children behind `hydrated`, every TanStack hook gates `enabled: accessToken !== null`). It's forward-looking plumbing for a future SSR/non-gated caller. Kept (not dead-code-removed) because removing it now means re-adding it later, and the cost is one if-branch.
- **W3** — no React-render test asserting the SIGNED_OUT/storage listener flow. Tests pin the store API surface and the regex contract; the wiring is unverified by automated tests. Rationale: vitest+jsdom for cross-tab supabase plumbing requires invasive supabase mocking that would itself be a maintenance burden. Manual two-tab smoke required before client hand-off.

### Phase 3 — UX honesty (M items, ~3 hours) ✅ SHIPPED 2026-05-11

RH-1, RH-2, RH-4 all closed in code.

**Audit**: single qa-auditor pass — verdict **PASS WITH WARNINGS** (1 critical, 5 warnings):

- **C1 (real bug caught)** — first-pass `SageStoredTurn.digest` typed `first_n_rows` (in-memory dispatcher shape). Backend `appendTurn()` at `sage-handler.ts:345` persists `first_5_rows`. Every restored thread would have rendered an empty Tabla tab with the correct rowCount badge — visible regression on the headline RH-4 feature. **Fixed**: type renamed to `first_5_rows`, hydrate reads the right key. Type-contract test pinned in `sage-threads-store.test.ts`.
- **W1** — `loadThread` catch removed the thread from local index on ANY error. Transient 5xx/network would silently erase the user's history. **Fixed**: branch on `ApiError.status === 404` (remove) vs other errors (surface retry hint in sidebar, leave entry alone).
- **W2** — filter chips not cleared when the X-axis field changed or drilled → stale chips matched nothing → blank chart. **Fixed**: `setAxis("x", ...)` auto-clears `filterPins` when field or geo/scian level changes.
- **W5 / chart+error not persisted** — `SageStoredTurn` no longer exposes `chart?` and `error?` fields; backend doesn't persist them, type now reflects reality.
- **R2** — `formatBinThresholds` produces "constante 5.0 (n=N)" instead of "≤5.0 · ≤5.0 · >5.0" when the sample is degenerate.
- **R5** — sidebar shows "máx. 20 hilos guardados" footer so the implicit cap is visible.
- **W3** (`onCellPin` hardcodes `axis: "x"` regardless of chart type — relies on data-shape coincidence) and **W4** (digest server-truncates to 5 rows; restored Tabla badge shows full count but only 5 rows) deferred — flagged in source comments. The "5 of N" UX nit is acceptable until users complain; W3 is a forward-compat concern for a not-yet-implemented heatmap chart type.

**Numbers**: 5 source files + 4 new test files + 2 new lib files, +1,612 / −91. Web tests 78 → **112 (+34)**: 18 quantiles + 5 LocustMode + 11 sage-threads-store. Backend test count unchanged at 948. Total vitest **1059 ✓**. Production build: SageMode chunk 9.3 → 13.9 KB (+4.6 for sidebar+store), main bundle 1.76 MB (unchanged; Phase 4 target). New `jsdom` devDep at both web/ and root/ for cross-dir vitest runs that pick up `web/src/lib/sage-threads-store.test.ts` (needs `@vitest-environment jsdom` for localStorage).

**Deferred (with justification)**:

- **W3** (`axis: "x"` hardcoded in cell-pin handler) — works for all currently-implemented chart types; matters only if a future `heatmap` impl emits y-axis names. Comment pinned at click site.
- **W4** (5-row server-side digest truncation vs row_count badge) — UX wart, not a bug. The badge is truthful; the table is a 5-row sample of the saved digest. Add "muestra" copy when a user reports confusion.
- **R3** (cross-tab StorageEvent sync for sidebar) — low-priority. Worst case: tab B doesn't see tab A's new thread until refresh.

### Phase 4 — Performance (M item, ~2 hours) ✅ SHIPPED 2026-05-11

RH-3 closed via a single new wrapper module (`web/src/lib/echarts-core.tsx`). Six consumers (LocustMode + 5 legacy dashboard charts) now import a tree-shaken `ReactECharts` instead of the bare `echarts-for-react` package. The wrapper calls `echarts.use([...])` exactly once with the curated registration list (5 chart types, 6 components, 2 features, 1 renderer).

**Bundle delta** (`dist/assets/index-*.js`):

| Metric | Before   | After    | Delta            |
| ------ | -------- | -------- | ---------------- |
| Raw    | 1,762 KB | 1,295 KB | −467 KB (−26.5%) |
| Gzip   | 560 KB   | 409 KB   | −151 KB (−27.0%) |

**Target framing**: plan goal was "main chunk under 1 MB raw". Landed at 1.30 MB raw — target missed by ~30%. **Gzip is the wire metric that actually matters** (Caddy serves with `encode zstd gzip` per `ops/Caddyfile.uncharted:27`), and on gzip we're well under 500 KB. Remaining raw bulk is React + supabase-js + TanStack Query + cmdk — not echarts. Further raw reductions would require code-splitting LocustMode itself (defer until a measured user-flow latency complaint exists).

**Audit**: single qa-auditor pass — verdict **PASS WITH WARNINGS** (0 critical, 3 warnings):

- **W1** (`SaludCobertura` `tooltip.axisPointer: { type: "shadow" }`) — uncertainty about whether the shadow indicator requires the standalone `AxisPointerComponent` in tree-shaken builds. **Fixed defensively** by adding `AxisPointerComponent` to the registration list (+~6 KB raw — well within the 467 KB headroom).
- **W2** (`TitleComponent` registered with zero consumers) — **dropped** to recover ~5 KB. Net of W1+W2 fixes: 1,297 KB → 1,295 KB.
- **W3** (heatmap toggle without impl in LocustMode) — pre-existing UI/impl gap from before Phase 4. `HeatmapChart` is registered (needed by `SectorGradeMatrix`) so the registration is correct; the UI/impl mismatch is unrelated to RH-3. Flagged on the v0.3.1 follow-up backlog.

**MapMode chunk unchanged** at 1.69 MB — that's deck.gl + maplibre-gl, out of RH-3 scope.

**Smoke-check matrix** (manual, before client hand-off):

- `/locust` → bar / scatter / line / treemap (cycle via Chart toggle)
- `/legacy/dashboard` → NationalTreemap, TopSectoresBar, DensidadPobrezaScatter (scroll legend), SaludCobertura (axisPointer shadow on hover), SectorGradeMatrix (visualMap)

**Numbers**: 1 new wrapper file + 6 consumers swapped (~12 LOC). 112 web tests still green (no test changes — chart-render assertions weren't part of the v0.3.1 test surface). Live at `index-CXJ1lew8.js`.
