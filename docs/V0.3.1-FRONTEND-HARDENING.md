**Anti-mission**: no new modes, no new data layers, no Datatur ingestion. Pure hardening.

### Phase 2 — Auth hardening (S items, ~2 hours) ✅ SHIPPED 2026-05-11

RH-10, RH-11, RH-12, RH-13 all closed in code. Cross-tab sign-out wires via supabase-js's native SIGNED_OUT broadcast (which already piggy-backs on storage events) + a belt-and-suspenders direct storage listener for externally-cleared tokens.

**Audit**: single qa-auditor pass — verdict **PASS WITH WARNINGS** (2 critical, 4 warnings):

- **C1 (real regression caught)** — first-pass `SAFE_QUERY` regex `/^[A-Za-z0-9._=&%-]*$/` rejected `! ~ * ' ( )`, which `encodeURIComponent` leaves un-encoded per RFC 3986 unreserved set. A user searching `Domino's` or `(Sucursal)` in `useSearch` would have gotten `bad_path 400` from the frontend before the call left the tab. **Fixed**: extended SAFE*QUERY to `/^[A-Za-z0-9.*=&%~!_'()-]_$/`. Regression test pinned in `web/src/api/client.test.ts` covering 7 representative passthrough chars.
- **C2 (comment drift)** — store comment claimed "listener short-circuits if session is already null" but listener calls cleanupLocal unconditionally. The flow is still correct (cleanupLocal is idempotent — that's the actual invariant), but the comment misled. **Fixed**.
- **W2** (positive/negative test mixed) — split into two `it` blocks.
- **R5** (regex DRY) — exported `SUPABASE_TOKEN_KEY_RE` from LoginGate; store test now imports the real one instead of recreating it.
- **W4** (token-refresh storage event amplification) — added inline comment explaining the cost shape.
- **W1** (session_loading is forward-looking plumbing) and **W3** (no LoginGate-render test) deferred — see deferreds below.

**Numbers**: 3 source files + 2 new test files, +247 / −20. 78 web tests (up from 52: +26 across `client.test.ts` and `store.test.ts`). Backend test count unchanged at 948. Total vitest 1026 ✓. Production build green; new index chunk `index-CdT-4Sgc.js` live; `/api/search?q=Domino%27s` returns 401 (auth) not 400 (frontend rejection), confirming C1 fix.

**Deferred (with justification)**:

- **W1** — `session_loading` code path has no production consumer today (LoginGate gates all children behind `hydrated`, every TanStack hook gates `enabled: accessToken !== null`). It's forward-looking plumbing for a future SSR/non-gated caller. Kept (not dead-code-removed) because removing it now means re-adding it later, and the cost is one if-branch.
- **W3** — no React-render test asserting the SIGNED_OUT/storage listener flow. Tests pin the store API surface and the regex contract; the wiring is unverified by automated tests. Rationale: vitest+jsdom for cross-tab supabase plumbing requires invasive supabase mocking that would itself be a maintenance burden. Manual two-tab smoke required before client hand-off.
