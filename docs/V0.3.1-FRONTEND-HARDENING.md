# v0.3.1 — Frontend Hardening Plan

**Status**: PLANNED (next session post-compact)
**Predecessor**: v0.3 demo shipped 2026-05-10 (`a0dad46`)
**Scope**: Close all R-tier deferreds from R2 (frontend UX) and C (auth flow) audits. No new mode features. No backend changes (server is closed).

## Mission

Take the v0.3 demo from "client-ready surface" to "client-handed-over surface". Every R-tier finding becomes either a fix or an explicit "won't fix because…" with a written reason. Bundle size drops. Cross-tab sign-out works. Filter pins do what they advertise.

**Anti-mission**: no new modes, no new data layers, no Datatur ingestion. Pure hardening.

RH-6, RH-7, RH-8, RH-9, RH-14, RH-15, RH-16, RH-17. Mostly one-line fixes or labels. Verify with the existing test suite.

### Phase 2 — Auth hardening (S items, ~2 hours)

RH-10, RH-11, RH-12, RH-13. Cross-tab sign-out is the headline. Test by opening two tabs and signing out in one.

### Phase 3 — UX honesty (M items, ~3 hours)

RH-1 (filter pins actually filter), RH-2 (legend palette adapts), RH-4 (saved threads). Each adds visible feature value vs the demo state.

### Phase 4 — Performance (M item, ~2 hours)

RH-3 (echarts tree-shake). Measure before/after with `npm run build`. Goal: main chunk under 1 MB.

### Phase 5 — Backend extension (M item, ~2 hours)

RH-5 multi-SCIAN bundle filter on `/tiles`. Backend touches:

- `src/api/handlers/tiles.ts` — accept `sector` as comma-separated, validate each, build IN(...) clause
- `src/api/handlers/tiles.test.ts` — new tests for multi-SCIAN
- Frontend: `web/src/modes/MapMode.tsx` handleBundlePick passes full `b.codes.join(',')` instead of just the 2-digit prefix

This is the ONLY backend touch in this hardening cycle. Everything else is frontend.

- Backend tests: ~990 passing (RH-5 adds tile multi-SCIAN tests, maybe +5)

- New modes (no "Reports" mode, no "Comparison" mode)
- Mobile responsive (still desktop-first)
- Per-user Sage budget (R-tier from audit B; deferred again unless a user count >5 lands)
- LLM-narrative quality tuning (Sonnet 4.6 is good enough for v0.3)
