# DENUE Analyzer — v0.3 Demo Plan: Three Entry Points

**Status**: PLANNING → BUILDING — 2026-05-10
**Scope**: Demo-grade UI for three modes (Locust, Mapview, Sage) backed by the v0.2.x closed backend.
**Sell-time**: provider-agnostic design lets a paying client swap the LLM backing for Sage without code changes.

---

## Vision

Three entry points landing on `/` (router redirects to `/locust` by default):

| Mode        | Anchor                                                                                                                       | Audience           |
| ----------- | ---------------------------------------------------------------------------------------------------------------------------- | ------------------ |
| **Locust**  | Single configurable chart at center with X/Y/Z axes + drill-down/up                                                          | The analyst        |
| **Mapview** | DENUE establishment points colored by bivariate/trivariate scale of polygon-level layers, anchored on a curated SCIAN bundle | The site planner   |
| **Sage**    | LLM that picks an endpoint (or drafts SQL), returns table + chart + narrative, with multi-turn refinement                    | The decision maker |

The three modes share: command palette, layer/field catalog, presets, auth gate, error boundary. Mode switching is instant (no full reload).

### Auto chart derivation rules

| X type      | Y type      | Auto chart                             |
| ----------- | ----------- | -------------------------------------- |
| categorical | numeric     | bar                                    |
| numeric     | numeric     | scatter                                |
| temporal    | numeric     | line                                   |
| categorical | categorical | heatmap                                |
| numeric_pct | numeric_pct | scatter (with diagonal reference line) |
| (any)       | (none)      | treemap of X by category               |

Override picker exposed as a dropdown in chart header.

### Z encoding

Always a green-to-red color gradient. Domain auto-computed from data with a manual override (lock min/max). Colorblind-safe palette: `viridis-inverted` as alternative shipped under a setting flag.

### Drill mechanic

- Each axis label shows the **drill level breadcrumb**: `Nacional ▸ Estado ▸ Muni` or `Sector ▸ Subsector ▸ Rama`.
- `−` (drill up) / `+` (drill down) controls operate on the axis the user clicks.
- "Drill down" on a category requires a selection — if Y is `pct_pobreza` at muni grain and X is `Sector`, clicking `+` after selecting a bar drills SCIAN one level deeper _within_ that selection.
- Cell click ≠ drill — it pins a filter chip (`SCIAN=46 AND pct_pobreza ∈ [25, 50]`).

### Default empty state

Empty canvas with two parallel calls-to-action:

1. **6 preset chips** along top: "Pobreza × Establecimientos", "Crimen tendencia × Mortalidad", "Cobertura salud × Población", "Subsidio SEDATU × Crédito CNBV", "Informalidad × Densidad económica", "Rezago × Farmacias controladas"
2. **Manual pickers**: `[Pick X]` `[Pick Y]` `[Pick Z]` rendered as large dotted-border placeholders.

### Layer catalog (Mapview-specific)

Subset of Locust fields restricted to those that make sense as a continuous color encoding:

- Muni-grain (~25 layers): `pct_pobreza`, `pct_pobreza_extrema`, `grado_rezago_social` (cast to ordinal), `homicidios_per_1k_year`, `trend_yoy_homicidios`, `tasa_mortalidad_cruda_per_1k`, `casos_dm2_promedio_mensual`, `casos_hta_promedio_mensual`, `farmacias_licenciadas_vigentes`, `valor_agregado_per_estab`, `monto_subsidiado_vivienda`, `monto_credito_comercial_total`, `brecha_credito_genero`, `pct_indigena_credito`, etc.
- AGEB-grain (~12 layers): `grado_rezago_social_ageb` (ordinal), `pct_sin_cobertura_salud`, `pobtot`, `pct_pobreza_ageb` (where available), `pct_pea_ageb`, etc.

### Bivariate color encoding

- 2 layers → 3×3 = 9-cell bivariate scale (Joshua Stevens palette)
- 3 layers → 3×3×3 = 27-cell trivariate scale (corner-cube legend)
- 1 layer → standard sequential green-to-red

Each polygon's `(layer1_bin, layer2_bin, layer3_bin)` triple maps to a color via lookup table. Each DENUE point inherits its containing polygon's color.

### Join arithmetic (locked: client-side)

1. Backend serves `/tiles/:z/:x/:y` (already exists) with each point carrying `cve_mun` + `cvegeo` + `scian_*` fields.
2. New backend route **`GET /analytics/layers/values?grain=muni|ageb&layers=A,B,C[&entidad=NN]`** returns:
   ```json
   {
     "grain": "muni",
     "layers": ["pct_pobreza", "homicidios_per_1k_year"],
     "values": {
       "01001": { "pct_pobreza": 25.4, "homicidios_per_1k_year": 0.12 },
       "01002": { ... }
     }
   }
   ```
3. Frontend computes per-polygon (binA, binB, binC) → color, applies to MapLibre paint expression keyed by polygon ID.
4. Layer swap = refetch `/layers/values` + recompute color table. Map geometry untouched.

Request:

```json
{
  "thread_id": "uuid-or-null-for-new",

```

```

event: table
data: {"columns":[...], "rows":[...]}

event: chart

event: narrative
data: {"text":"En Veracruz, los municipios con..."}

event: usage

data: {"turn_id":"uuid"}
```

### Provider abstraction (THE commercial enablement)

`src/api/sage/providers/`:

- `provider.ts` — interface (routeAndDraft, writeNarrative, writeNarrativeStream, countTokens, normalizeUsage)
- `anthropic.ts` — uses `@anthropic-ai/sdk` for the demo
- `openai-compatible.ts` — uses `fetch` against any OpenAI v1-compatible endpoint
  - Covers: Groq, Fireworks, Together, DeepInfra, OpenRouter, vLLM-hosted, Ollama, llama.cpp server, Mistral, OpenAI itself
- `index.ts` — factory dispatching by `SAGE_PROVIDER` env

Tool-calling format normalization happens in the provider boundary. Each provider returns a `RouteOutput` of shape:

```typescript
{
  kind: "endpoint" | "sql";
  endpoint?: { name: string; params: Record<string, string|number> };
  sql?: string;
  confidence: number;
  reasoning: string;
}
```

### SQL safety rail (four layers)

1. **Postgres role `denue_sage`** — `GRANT SELECT` only on allowlisted views/MVs (NOT raw `establecimientos`). Created via migration in `scripts/sage-role.sql`.
2. **Parser**: pg-query-emscripten parses the SQL. Reject anything that isn't `SELECT` / `WITH … SELECT`. No DML, no DDL, no `pg_*` functions, no functions outside `{lower, upper, count, sum, avg, min, max, coalesce, nullif, round, abs}`.
3. **Plan budget**: `EXPLAIN (FORMAT JSON)` first; reject if `Total Cost > 5e6` or any node is `Seq Scan` over `establecimientos`.
4. **Runtime guards**: `statement_timeout=8000`, forced `LIMIT 5000`, payload cap 2 MB.

Errors return structured codes (`SQL_PARSE_FAIL`, `SQL_PLAN_TOO_EXPENSIVE`, `SQL_FORBIDDEN_TABLE`, `SQL_TIMEOUT`) so the LLM can self-correct on next turn.

### Multi-turn state shape

Per-turn record:

```typescript
interface Turn {
  question: string;
  route: { kind: "endpoint"|"sql"; endpoint?: {...}; sql?: string; };
  digest: {
    columns: string[];
    row_count: number;
    first_5_rows: unknown[];
    numeric_stats: Record<string, {min:number,max:number,mean:number}>;
  };
  narrative: string;
  usage: { input_tokens: number; output_tokens: number; cost_usd: number; };
}
```

Threads persisted in a new table `sage_threads(thread_id UUID, turns JSONB[], created_at, updated_at)` for resume. Last 5 turns sent as digests in the prompt; turns 6+ get summarized into a single sentence by Haiku-equivalent (cheap router model).

### Models for the demo

- `SAGE_PROVIDER=anthropic`
- `SAGE_MODEL_ROUTER=claude-sonnet-4-6` (handles route picking + SQL drafting)
- `SAGE_MODEL_NARRATIVE=claude-sonnet-4-6` (handles prose)
- No cost cap for the demo. Cost telemetry instrumented anyway.
