## v0.2.3-A — EDR mortalidad

### Qué agrega

Mortalidad municipal: defunciones totales, tasa cruda por 1k habitantes, top causas (CIE-10), mortalidad infantil. Cierra el cuadrante "salud poblacional" (CLUES = capacidad instalada; EDR = outcomes).

### Source

- **Programa:** Estadísticas de Defunciones Registradas (EDR), INEGI
- **Anteriormente:** SINAIS mortalidad — INEGI absorbió la serie
- **Página:** https://www.inegi.org.mx/programas/edr/
- **Granularidad:** registro individual; campos `ent_resid` (entidad de residencia) + `mun_resid` (municipio de residencia 1-3 dígitos sin zero-pad). Construir `cve_mun = LPAD(ent_resid,2,'0') || LPAD(mun_resid,3,'0')`.
- **Variables clave:**
  - `causa_def` — código CIE-10 (5 chars, e.g. `I219`)
  - `edad_anios`, `sexo` — demografía básica
  - `ano_ocur`, `mes_ocur`, `ano_regis` — fechas
  - `lista_mex` — agrupador INEGI (215 categorías)

### Operator delivery (browser required)

INEGI bloquea descargas headless con un interstitial JS. Operador descarga vía navegador:

1. Abrir https://www.inegi.org.mx/programas/edr/#datos_abiertos
2. Bajar **"Conjunto de datos abiertos: 2023"** (ZIP, ~80-150 MB CSV interno)
3. Si 2023 aún no publicado para descarga abierta, usar 2022 (más reciente listado en página al 2026-05-05)
4. Soltar en `data/edr/conjunto_de_datos_defunciones_registradas_YYYY_csv.zip`
5. Avisar para correr el loader

Loader ya quedará escrito (ver siguiente sección) — solo espera el archivo.

### Schema target (Postgres)

```sql
CREATE TABLE inegi_edr_defunciones_raw (
  id BIGSERIAL PRIMARY KEY,
  ano_ocur INT NOT NULL,        -- 1985-2099 CHECK
  mes_ocur INT NOT NULL,        -- 1-12 CHECK
  ent_resid TEXT,               -- 2-char zero-pad
  mun_resid TEXT,               -- 3-char zero-pad
  cve_mun TEXT,                 -- ent_resid || mun_resid, NULL when no residencia
  sexo INT,                     -- 1=hombre 2=mujer 9=NE
  edad_anios INT,
  causa_def TEXT,               -- CIE-10 code
  lista_mex INT                 -- agrupador INEGI
);
CREATE INDEX inegi_edr_cve_mun_idx ON inegi_edr_defunciones_raw (cve_mun);
CREATE INDEX inegi_edr_ano_idx ON inegi_edr_defunciones_raw (ano_ocur);
```

### Mat-view target

```sql
CREATE MATERIALIZED VIEW mv_mortalidad_municipal_yearly AS
SELECT
  cve_mun,
  ano_ocur AS ano,
  COUNT(*)::bigint AS total_defunciones,
  COUNT(*) FILTER (WHERE edad_anios < 1)::bigint AS defunciones_menores_1ano,
  COUNT(*) FILTER (WHERE causa_def LIKE 'I%')::bigint AS causas_circulatorio,
  COUNT(*) FILTER (WHERE causa_def LIKE 'C%')::bigint AS causas_neoplasias,
  COUNT(*) FILTER (WHERE causa_def LIKE 'V%' OR causa_def LIKE 'W%' OR causa_def LIKE 'X%' OR causa_def LIKE 'Y%')::bigint AS causas_externas,
  COUNT(*) FILTER (WHERE causa_def LIKE 'E%')::bigint AS causas_endocrinas
FROM inegi_edr_defunciones_raw
WHERE cve_mun IS NOT NULL AND LENGTH(cve_mun) = 5
GROUP BY cve_mun, ano_ocur;

CREATE INDEX mv_mortalidad_yearly_cve_mun ON mv_mortalidad_municipal_yearly (cve_mun);
CREATE INDEX mv_mortalidad_yearly_ano ON mv_mortalidad_municipal_yearly (ano);
```

### Endpoints target (2 nuevos)

```
GET /analytics/mortality-summary?entidad=XX[&ano=YYYY]
  Per-municipio mortality profile: total + tasa per 1k pop (joining
  Censo 2020) + top causes + infant mortality. Mat-view-first w/ live
  fallback (mismo patrón que risk-summary).

GET /analytics/mortality-trend?cve_mun=NNNNN
  Annual mortality time series 2010-latest, single municipio.
```

### Loader scaffold (queue)

`scripts/load-edr.ts`:

- Reuse `RnidInput` discriminated-union pattern from `scripts/load-sesnsp.ts`
- Stream CSV via `head -1` for header + `iconv -f WINDOWS-1252 -t UTF-8 | tail -n +2 | tr -d '\r'` for body (encoding TBD; older años son W-1252, recientes UTF-8 — auto-detect via header BOM)
- COPY into raw table; derive `cve_mun` via `UPDATE` after COPY (NULL para muertes sin residencia válida)
- Refresh-matviews script: añadir `mv_mortalidad_municipal_yearly`

### Tests target

- ~10 nuevos tests en `analytics.test.ts` cubriendo ambos endpoints (success / 400 validation / 401 / mat-view fallback / malformed JSON)
- Loader unit tests para `cve_mun` derivation y header validation

## v0.2.3-B — Datatur (deferred)

### Qué bloquea

INEGI publica los 70 destinos turísticos sin un mapping oficial a `cve_mun`. ~50/70 son municipio único (Acapulco = `12001`, Guadalajara = `14039`, etc.); ~20 son zonas hoteleras sub-municipales o agregados multi-municipio (Riviera Maya, Zona Hotelera CDMX, etc.).

### Tarea bloqueante (~1 día)

Construir `data/datatur/destinos_cve_mun_crosswalk.csv` con columnas:

```
destino_id,destino_nombre,cve_mun_principal,cve_mun_extra (json array),notas
```

Fuente: lista oficial Datatur destinos × catálogo INEGI municipios + revisión manual con Wikipedia/notas de campo. Una vez existente, el loader es directo (mensual desde Datatur datos abiertos).

### Cuándo activar

Después de operator interest en señal turística para casos retail/F&B/hospitality. No-blocking para uso general del analyzer.

## v0.2.3-C — ENOE / ENIGH (skipped)

Documentado aquí para que futuros operadores no re-investiguen.

**ENOE**: muestra trimestral diseñada para inferencia nacional + 32 entidades + 39 ciudades autorrepresentadas. Microdata lleva `mun` pero INEGI's notas de microdatos prohíbe explícitamente inferencia sub-ciudad. Si se necesita empleo a nivel municipal, usar CE 2024 (`personal_ocupado_total`) — ya cargado.

**ENIGH**: muestra bienal n=105,718 hogares; ~3,300/estado. Insuficiente para inferencia municipal por orden de magnitud. Variables (ingreso, gasto, deciles) ya cubiertas a nivel municipal por CONEVAL pobreza.

Si se necesitan ENOE/ENIGH como **calibradores estatales** (no como filas joinadas), el patrón sería un parameter table separado, no una columna en `inegi_municipios_full` u otra agregación municipal. Diferido sin fecha.
